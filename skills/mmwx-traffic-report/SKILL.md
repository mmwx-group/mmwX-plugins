# 流量巡检报告(妙妙屋X)

目标:快速给出可读的流量概览与异常名单。全程只读,安全。

## 步骤

1. **总体概览**:`traffic_summary` 取跨服务器聚合用量(已用/限额等)。
2. **用户维度**:`user_list` 取全部用户及其套餐/配额;对关注的用户用 `traffic_user_detail`(传 username)拿明细。
3. **(可选)服务器维度**:`server_list` 看各服务器状态;需要单台明细可经只读流量接口补充。
4. **汇总成报告**,建议包含:
   - 总用量 / 总限额、整体使用率。
   - 用量 Top N 用户(名称、已用、限额、使用率)。
   - **风险名单**:使用率 ≥ 80%(临近超额)与 ≥ 100%(已超额)的用户。
   - 离线/异常服务器(若有)。
5. **给建议**(不自动执行):如对超额用户提示可 `user_set_limits` 限速或 `package_assign` 调套餐——但这些写操作要先征得用户同意再做。

## 注意
- 本技能默认只读,不要在未确认的情况下改用户/套餐。
- 计费流量受套餐 `traffic_mode`(oneway=×1 / twoway=×2)影响,报告里注明口径。
